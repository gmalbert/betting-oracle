Machine Learning (ML) models


XGBoost

Why: Fast gradient boosting with strong tabular performance and good missing-value handling.

Insights: Feature importance, SHAP explanations, and regularization pathways for model stability.

LightGBM

Why: High-speed, memory-efficient boosting with native categorical support and large dataset handling.

Insights: Leaf-wise tree growth, model size tuning, and fast hyperparameter sweeps.

Random Forest

Why: Stable, interpretable ensemble baseline with built-in out-of-bag scoring.

Insights: Permutation importance, tree structure profiling, and low variance behavior on noisy tabular data.

CatBoost

Why: Handles categorical features natively and reduces need for manual preprocessing.

Insights: Ordered boosting for overfitting resistance and built-in feature and permutation importance.

Neural Networks / LSTM

Why: Captures complex non-linear patterns in time-series and sequence features.

Insights: Feature embeddings, attention/sequence activations, and hybrid model stacking with tree learners.

Ensembles and Calibration

Why: Combine multiple models and calibrate probabilities for more reliable edge detection.

Insights: Stacked models, bagging/boosting hybrid, probability calibration (Platt scaling, isotonic).

Repo-specific Models

Specialized model types used in each project, from logistic regression to ELO and player prop scoring.

  • Logistic Regression: value probability modeling, manager/referee effect analysis (premier-league).
  • ELO + custom ratings: team/player form and matchup strength (all sports models).
  • Poisson/Score-based regressions: player prop and goal prediction pipelines (hockey, soccer).
  • Gradient Boosting variants: domain-specific feature-discrete handling (gridiron, tennis, golf).
  • Time-series growth: sequential injury trend and live tracking models (nba-predictions).
